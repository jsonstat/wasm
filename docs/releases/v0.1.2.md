## jsonstat-wasm v0.1.2

Performance and memory optimizations. **No public API changes and no changes to the shape of returned values** — this release is a drop-in replacement for `0.1.1`.

### Parsing
- **Early-exit class detection.** `class` is now found with a single short-circuiting scan that stops as soon as the discriminator is seen, instead of lexing the whole document (including a potentially large `value` array) before the real parse.

### Value access (`Data()`, `Datum()`, `value`)
- **Fewer per-cell allocations in `Data()`.** The all-cells and slice paths build `{value, status}` results via a struct with static field names, eliminating the two per-cell map-key `String` allocations the previous `json!` macro performed. Emitted JS objects are unchanged.
- **Allocation-free sparse lookups.** Sparse `value` objects and object-form `status` are now indexed with a stack-buffer decimal key instead of allocating a `String` per cell.

### Transformations (`Dice()`, `Transform()`, `Unflatten()`)
- **Incremental, allocation-light iteration.** `Dice()` walks the kept coordinate space with an odometer (no full materialization), clones each dimension once, and avoids cloning the full `value`/`status`. `Transform()`/`Unflatten()` use stride-based flat indexing and avoid redundant per-cell allocations.

### Memory
- Internal cleanups reduce transient allocations across the parse and bulk-access paths. `ToJSON()` serializes from the typed model.

### Verification
- Full host test suite, `clippy`, `wasm32-unknown-unknown` check, and the release `wasm-pack` build all pass.
