## jsonstat-wasm v0.4.1

A **packaging** release: the published JavaScript is now minified (with source
maps), and a stale version reference in the docs is corrected. No API or
behavior change — every parse route produces byte-identical output.

### Minified artifacts

The facade and the WASM glue were previously shipped as readable, non-minified
ESM. v0.4.1 runs both through esbuild ([`scripts/build.sh`](../scripts/build.sh),
`--minify --sourcemap`) so the npm/CDN payload is substantially smaller, while
shipping the readable source and source maps alongside for debugging. No
`--format` flag is passed, so esbuild preserves each wasm-pack target's native
module system: ESM for `web` (what npm/CDN ships), CommonJS for `nodejs` (used
by the local Node test harnesses in `.idea/test/`).

**Naming convention.** The bare name stays the default (minified) entry, and
the readable non-minified source is published as a `.max.js` sibling:

| File | What it is |
|---|---|
| `jsonstat.js` | facade, **minified** (default entry: `main` / `module` / `exports["."]`) |
| `jsonstat.max.js` | facade, readable source (debugging) |
| `jsonstat.js.map` | facade source map (→ `jsonstat.max.js`) |
| `jsonstat_wasm.js` | low-level glue, **minified** (`exports["./glue"]`) |
| `jsonstat_wasm.max.js` | low-level glue, readable source |
| `jsonstat_wasm.js.map` | glue source map (→ `jsonstat_wasm.max.js`) |

**Sizes** (isolated harness against the v0.4.0 glue):

| Artifact | Before | After | Reduction |
|---|---|---|---|
| facade | 14.5 KB | 2.1 KB | ~86% |
| glue | 55.5 KB | 20.0 KB | ~64% |

**Semantics preserved.** `--minify` renames only internal locals — the module's
exported bindings (`JSONstat`, `init_panic_hook`, `version`, `DimensionInstance`,
`initSync`, the default `init`) are kept — and, with no `--format` flag, esbuild
preserves the glue's native ESM (web: `export class JSONstat`, `import.meta.url`
locating `jsonstat_wasm_bg.wasm`) or CommonJS (`nodejs`: `module.exports`). So
the facade's named imports still resolve against the glue's exports unchanged,
and the `./glue` subpath keeps working. Verified end-to-end: the minified facade
imports `{JSONstat, version, init_panic_hook}` + default from `./jsonstat_wasm.js`,
the web glue exports them with `import.meta.url` intact, and the 60-check Node
parity harness ([`verify-fromobject.mjs`](../.idea/test/verify-fromobject.mjs))
passes on the minified node target — fast-path, serde, and string parses are
byte-identical.

### Stale version reference corrected

[`jsonstat.js`](../jsonstat.js) and [`docs/INSTALL.md`](../docs/INSTALL.md)
still pinned a `@0.3.0` CDN URL in their examples after v0.4.0. Both now read
`0.4.1` (the version they actually ship in). Historical `v0.3.0` *feature
attributions* (e.g. "Since v0.3.0 …", the `@see …/v0.3.0.md` link) are
intentionally left as-is.

### What's unchanged

- **No API change.** Same `JSONstat(string|object|url)`, same surface.
- **No behavior change.** Minification is semantics-preserving; the Rust/WASM
  binary is unchanged from v0.4.0.
- The release build profile (`opt-level = 3`, `lto = "fat"`,
  `codegen-units = 1`) is unchanged.

### Verification

- Isolated build harness confirmed: facade import bindings and glue exports
  preserved through minification; both source maps emitted; `pkg/package.json`
  `files` and `exports` map correctly include the new artifacts.
- `esbuild@0.25.0` (pinned in [`scripts/build.sh`](../scripts/build.sh)) and
  the reordered Node-setup step in CI ensure the minify step is reproducible
  locally and in [`release.yml`](../.github/workflows/release.yml).

### Upgrading from v0.4.0

Nothing to do — drop-in. If you depend on the exact bytes of `jsonstat.js`,
note it is now minified; the readable source is `jsonstat.max.js` and the map
is `jsonstat.js.map`.
